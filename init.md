Задача разработаь агента для подготовки статей. Статьи нужны для разных площадодк, для  
 разной аудитории. Система должна иметь UI позволять регистрироваться и  
 авторизовываться. Пользователь может завести несколько профилей площадок, с указанем  
 основных параметров публикаций: формат, стиль, аудитория, объем, требования к разметке,  
 дополнительные текстовые требования (как часть промпта)  
 Агент (система) должна помогать пользователю определиться с темой публикации, и вместе  
 разработать каркас (топики, связи, цель, основная идея и так далее). Агент должен уметь  
 применять распространенные методики написания статей в зависимости от цели.  
 После определения плана статьи, агент должен предлагать(если уместно) поискать  
 источники. ВЕроятно путем гипотезы о том, какой материал может быть полезен, с  
 последующим генерацией поисковых фраз, поиска, извлечения и изучения материалов.  
 В некотрых случаях агенту может быть дано задание на рерайт существующей статьи.  
 Кроме подготовки текста статьи агент должен умень предложить декорирование статьи -  
 вставки, блоки, цитаты, а также помочь с выбором иллюстрации. Иллдюстрации могут быть -  
 главной иллюсьтрацией к статье так и иллюстрации по тексту.

Задача много, и агент должен для разных задач использовать разные по силе модели.

1. Умную думающую модель для интеллектуальных задач
2. быструю эффективную модель для суммаризации, поиска, классификации и тому подобного
3. модель для поиска (Sonar Pro)
4. модель для генерации изображения (NanoBanana и Image 2 от openai)

все взаимодействие должно происходить через openrouter api.  
 должен быть контроль бюджета. Пока именно на уровне фиксации, но позже и на уровне  
 ограничений  
 отдельные части пайплайна должен быть оформлены в коде таким образом, чтобы удобно было  
 построить eval системы для отработки отдельных участков  
 Должно быть логирование всего процесса в jsonl файлы  
 Надо продумать пользователький интерфейс работы со статьей. Желательно, чтобы было  
 основное поле с материалами, чат с агентом, где было бы видно, что делает агент, какие  
 задачи выполняются и на каком мы этапе.  
 в результате получившуюся статью можно скачать в md, html, docx, pdf  
 при работа с изображениями нужно использовать практики продвинутого управления промптом  
 (json формат) и должен быть этап пререндера, когда пользователю показывают несколько  
 вариантов изображений на выбор.  
 также должна быть возможность выбарть изображение из бесплатного стока. НЕ всегда нужно  
 именно рендер.

Это плохо структурированный но досаточно подробный план продукта. Твоя задача сделать  
 из этого плана PRD + ARCHITECTURE + IMPLEMENTATION_PLAN. + написать два промпта. один  
 для планирования следующего эпика, второй для имплементации задач текущего эпика.  
 То есть агент планирования ищет в IMPLEMENTATION_PLAN очередной PLANING_CHECKPOINT и  
 планирует следующий участок иимплементации. а потом в цикле запускается агент с  
 промптом имплементации и ему на вход дают одну за другой задачи из уже запланированных.
